Diabetes Detection
Tell Me More



Our goal is to be able to predict whether a person has diabetes or not. The user will fill out a few questions about themselves, such as blood pressure and BMI, and our product will take that input and predict whether that person is diabetic or not.

Data

Go to our data set here

Original Set

We used a data set from Kaggle that included 8 independent features and 1 dependent feature. The outcome was dependent on pregnancies, BMI, glucose, blood pressure, skin thickness, age, insulin, and diabetes pedigree.

Cleaning Data

The data set did not initially contain any missing values. However, some of the independent features had zero values, which made no sense. If one's blood pressure was 0, how could they be alive? We therefore changed all of those zero values to null values. Then, we replaced those null values with the mean of that certain feature to have a more accurate data set.

Final Data

Once our data was cleaned, we were left with a more realistic and accurate data set that we could then input into our machine learning model.

Story

This section shows the history of the whole process, from choosing the data set to figuring out the best machine learning model for the data set.

  • The Beginning:

    Why We Chose Diabetes

    Diabetes affects more than 463 million people in the world. 37.3 million of them are Americans. Even though approximately 1 in 10 Americans have diabetes, 1 in 5 of them don't even know they have it. Thus, the purpose of our research is to be able to predict whether someone is truly healthy or is diabetic.

  • Diving Deeper:

    Data

    The data set chosen included information that could affect the outcome of diabetes, such as: pregnancies, glucose, blood pressure, skin thickness, insulin, BMI, age, and diabetes pedigree function.

  • Making Connections:

    Correlation Within the Data

    Once the data set was finalized, we plotted the data in various graphs to make a connection between the data points. One interesting fact we discovered was that some of the features didn't affect the outcome of diabetes - such as skin thickness, insulin, and blood pressure. However, other features - such as glucose and BMI - did affect the outcome.

  • Modeling the Data:

    Machine Learning Models

    We decided to use 4 machine learning models to help analyze/predict outcomes using data from the diabetes data set. We used Random Forest, KNN, Neural Networks, and SVM. The KNN model was the best, as it had a reasonable computation time and provided the best results. So, we chose the KNN model as our main model.

  • KNN Model:

    Results

    The KNN model had the highest scores for every main category we were looking at: accuracy, precision, and recall. The KNN model had an accuracy rating of 81.17%, a precision rating of 83.18%, and a recall value of 85.58%. All of those ratings were the best of all the models we tested.
